# Lógica Cero | Portafolio Odoo & Integraciones

## 1. Introducción y Propuesta de Valor
Lógica Cero es un emprendimiento de programación Odoo fundado por dos programadores apasionados con base en Maracaibo, Zulia; capaces de atender proyectos nacionales e internacionales. Nacemos para superar las limitaciones de los partners tradicionales: hacemos realidad integraciones y personalizaciones que otros no pueden o no quieren abordar.

Eslóganes:
- "Lo que puedan imaginar, lo podemos crear."
- "Hecho con lógica, cero complicaciones."

Transformamos ideas en soluciones funcionales, optimizando procesos y conectando Odoo con ecosistemas externos de forma limpia, auditable y escalable.

## 2. Génesis y Experiencia del Equipo
Evolución:
- Consultoría funcional: comprensión profunda de procesos reales.
- Liderazgo técnico de proyectos: visión estratégica end-to-end.

Historial comprobado:

## 3. Servicios y Demostración de Capacidades
Enfoque principal: Desarrollo avanzado e Integraciones de alto valor.

Casos representativos (resúmenes):
- Finanzas Estratégicas (Odoo Bridge): Automatización de declaraciones y visualización dual entre sistemas.
- Visibilidad Financiera (Triangulación de Fondos): Tablero tipo CRM para monitorear compras en tránsito y capital comprometido.
- Logística Especializada (Surtex / TruckFlow / MRW Connect): Módulos para requisiciones de mantenimiento, control de viajes y tracking/cotización de envíos.
- Integraciones multi-API: Conectores resilientes con manejo de reintentos, control de latencia y trazabilidad.
(Nota: Casos ampliados disponibles en la web como prueba de concepto.)

## Información Técnica del Repositorio

Estructura:
- index.html: Página principal.
- assets/css/style.css: Estilos base.
- assets/js/main.js: Lógica (carga dinámica, filtros, tema).
- data/*.json: Fuentes de datos (proyectos, integraciones).

Deploy rápido (Netlify):
1. Subir repositorio.
2. Configurar build estático.
3. Activar formularios (Netlify Forms).

Personalización:
- Agregar proyectos en data/projects.json.
- Ajustar paleta en :root (style.css).
- Añadir imágenes en assets/img y referenciar en tarjetas.

Mejoras futuras:
- Internacionalización (i18n).
- Modo offline (Service Worker).
- Tests de accesibilidad (Lighthouse / axe).
